# Agent workflow: update or add site videos

Use this runbook when the artist drops new masters (`.mov` / `.mp4`) and wants them
live on [finalbattle.video](https://finalbattle.video).

## Architecture (read first)

| What | Value |
|------|-------|
| R2 bucket | `finalbattleiran-media` |
| Public CDN | `https://videos.finalbattle.video` |
| Short-form R2 keys | `videos/<slug>.mp4`, `videos/<slug>-fa.mp4` |
| Local encodes (dev) | `web/public/videos/` |
| Content metadata | `web/src/content/videos/<slug>.json` |
| Thumbnails (committed) | `web/public/videos/thumbs/<slug>.webp`, `<slug>-fa.webp` |

**Important:** Production resolves root-relative `videoSrc` values (e.g.
`/videos/foo.mp4`) via `PUBLIC_VIDEO_BASE_URL`. The code default is
`https://videos.finalbattle.video` (`web/src/consts.ts`). The Cloudflare Pages
dashboard must set the same value so production Functions and pages agree.

**Filename convention:** clean slug names, no date suffix. Each batch overwrites
the same R2 keys so URLs (and shares) stay stable across re-cuts. English is
`<slug>.mp4` / `<slug>.webp`; Persian is `<slug>-fa.mp4` / `<slug>-fa.webp`.

MP4 masters and encodes are **gitignored**. Only JSON metadata and WebP thumbnails
get committed.

### Prerequisites

- **ffmpeg** on `PATH` (Windows: `winget install Gyan.FFmpeg`)
- **Node.js 22+** and `npm install` in `web/`
- **Wrangler logged in** to the Cloudflare account that owns the bucket:
  `cd web && npx wrangler whoami`
- Read access to the source masters folder (e.g. `D:\FinalBattle\Output\...`)

## Workflow A — Replace existing videos from a source folder

Typical case: new DaVinci exports land in a dated output folder and map to videos
already on the site.

### 1. Map sources to slugs

Match each master to the `<slug>` of a `web/src/content/videos/*.json` file.
Example mapping (July_12 batch):

| Source master | Slug | Site MP4 |
|---------------|------|----------|
| `Captive_Nation_English_July_12.mov` | `captive-nation` | `captive-nation.mp4` |
| `Captive_Nation_Farsi_July_12.mov` | `captive-nation` (fa) | `captive-nation-fa.mp4` |
| `Return_To_Democracy_English_July_12.mov` | `returning-to-democracy` | `returning-to-democracy.mp4` |
| `Return_To_Democracy_Farsi_July_12.mov` | `returning-to-democracy` (fa) | `returning-to-democracy-fa.mp4` |
| `Transitional_Leader_English_July_12.mov` | `transitional-leader` | `transitional-leader.mp4` |
| `Transitional_Leader_Farsi_July_12.mov` | `transitional-leader` (fa) | `transitional-leader-fa.mp4` |
| `Bright_Future_English_July_12.mov` | `the-nation-has-a-future` | `the-nation-has-a-future.mp4` |
| `Bright_Future_Farsi_July_12.mov` | `the-nation-has-a-future` (fa) | `the-nation-has-a-future-fa.mp4` |
| `By_The_People_English_July_12.mov` | `by-the-people` | `by-the-people.mp4` |
| `By_The_People_Farsi_July_12.mov` | `by-the-people` (fa) | `by-the-people-fa.mp4` |
| `Far_From_Home_English_July_12.mov` | `far-from-home` | `far-from-home.mp4` |
| `Far_From_Home_Farsi_July_12.mov` | `far-from-home` (fa) | `far-from-home-fa.mp4` |

### 2. Encode + thumbnail

Edit `web/scripts/encode-videos.ps1`: set `$SourceDir` and the `$Jobs` array
(`{ In = "<source file>"; Out = "<slug>.mp4" }`). The script transcodes every
master to a web-capped bitrate and extracts a 9:16 WebP poster in one pass.

```powershell
cd web
npm run encode:videos
```

DaVinci masters here run 12–27 Mbps H.264, so they are **always transcoded**
(not remuxed) down to `maxrate 3500k`. Encoding is fast (~6× realtime,
`-preset medium`), so the old "remux only" shortcut no longer applies.

### 3. Upload to R2

Edit `web/scripts/upload-videos-r2.ps1` if slugs changed (the `$Uploads` list and
the `$Stale` cleanup list), then:

```powershell
cd web
npm run upload:videos
```

Each upload uses `--remote`. The script also deletes superseded objects so the
bucket doesn't accumulate cruft.

Manual one-off:

```powershell
npx wrangler r2 object put finalbattleiran-media/videos/<slug>.mp4 `
  --remote --file public/videos/<slug>.mp4 --content-type video/mp4
```

### 4. Update metadata

In each `web/src/content/videos/<slug>.json`:

- `durationSeconds` — probe with
  `ffprobe -v error -show_entries format=duration -of csv=p=0 public/videos/<slug>.mp4`
  and round to nearest integer.
- `script` / `title` / `description` / `context` if the edit changed copy.
- `videoSrc` / `thumbnailUrl` only if you renamed the slug.

### 5. Build + verify

```powershell
cd web
npm run build          # runs gen:videos + gen:products, then astro build
npm run dev            # spot-check http://localhost:4321/videos/<slug>/
```

Confirm the CDN object matches the local encode:

```powershell
Invoke-WebRequest -Uri "https://videos.finalbattle.video/videos/<slug>.mp4" -Method Head
```

### 6. Ship

Commit metadata + thumbnails (never `*.mp4`). Pushes to `main` redeploy Pages.

- `web/src/content/videos/*.json`
- `web/public/videos/thumbs/*.webp`
- `web/functions/api/_videos.json` (regenerated by `gen:videos`)

## Workflow B — Add a brand-new video

### 1. Create content JSON

Add `web/src/content/videos/<slug>.json` (copy an existing file as a template).
Until the copy is ready, mark it **`"draft": true`** so it is hidden from the
production library and routes (still visible in `npm run dev`).

```json
{
  "series": "prosperity-project",
  "category": "truth",
  "boostTier": 3,
  "publishedAt": "2026-07-09",
  "order": 5,
  "featured": false,
  "draft": true,
  "tags": ["TODO"],
  "languages": {
    "en": {
      "title": "Video Title",
      "videoSrc": "/videos/<slug>.mp4",
      "description": "One-line card blurb.",
      "durationSeconds": 42,
      "thumbnailUrl": "/videos/thumbs/<slug>.webp",
      "script": "Full transcript…",
      "context": "Background essay for the detail page…",
      "platforms": {
        "x": "https://x.com/…/status/…",
        "tiktok": "https://www.tiktok.com/@…/video/…",
        "youtube": "https://youtube.com/shorts/…"
      }
    },
    "fa": { "…": "add the Persian cut when ready; same shape as en" }
  }
}
```

Notes:

- A video appears in a locale's library **only** if that locale exists under
  `languages` **and** `draft` is not `true` in production.
- Persian filenames convention: `<slug>-fa.mp4` and `<slug>-fa.webp`.
- `boostTier`: `1` = ad-safe, `2` = caution (ad-safe), `3` = organic-only. Unset
  defaults to `3` with a build warning. Keep new/unreviewed clips at `3`.
- For long-form (> ~2 min), set `"format": "long"` on the entry root.

### 2. Encode, upload, thumbnail

Add rows to `encode-videos.ps1` and `upload-videos-r2.ps1`, then run both (steps
2–3 in Workflow A). Flip `draft` off once copy + platform links are filled in.

## Encoding rules

Masters are transcoded to a web-capped bitrate (phones/social don't need 20 Mbps).

**Short-form** (the common case):

```powershell
ffmpeg -hide_banner -y -i master.mov `
  -c:v libx264 -profile:v high -pix_fmt yuv420p `
  -preset medium -crf 23 -maxrate 3500k -bufsize 7000k `
  -c:a aac -b:a 128k -movflags +faststart `
  public/videos/<slug>.mp4
```

**Thumbnail** (9:16 WebP poster; `-ss 3` for `"format": "long"`, else `-ss 1.5`):

```powershell
ffmpeg -hide_banner -loglevel error -ss 1.5 `
  -i public/videos/<slug>.mp4 -frames:v 1 `
  -vf "scale=540:960:force_original_aspect_ratio=increase,crop=540:960" `
  -c:v libwebp -quality 82 -y public/videos/thumbs/<slug>.webp
```

`encode-videos.ps1` does both in one pass. To regenerate thumbnails from the
hosted files instead, use `npm run gen:r2-thumbs` (reads every JSON, both locales).

## Pitfalls

| Symptom | Cause | Fix |
|---------|-------|-----|
| `moov atom not found` on CDN or ffprobe | Upload interrupted or encode killed mid-write | Re-encode locally, verify with `ffprobe`, re-upload |
| Site still plays old video in production | Pages env `PUBLIC_VIDEO_BASE_URL` wrong | Set to `https://videos.finalbattle.video` in dashboard |
| Wrangler upload succeeds but URL 404s | Wrong R2 key path | Shorts → `videos/<slug>`; check the `videos/` prefix |
| Local dev works, prod broken | MP4 only on disk, not uploaded | Run `npm run upload:videos` |
| New skeleton video shows placeholder text live | `draft` not set / removed too early | Keep `"draft": true` until copy is final |

Always pass **`--remote`** to `wrangler r2 object put`/`delete` when touching
production objects.
